# 小红书四模式执行商业策划案（最终优化版）

## 0. 一页结论
- 核心目标：在 90 天内跑通“内容获客 -> 私信分流 -> 分层成交 -> 复购增长”的稳定商业闭环。
- 战略选择：先打透 `模式一（考公考研）+ 模式二（职场副业）` 作为现金流引擎，再用 `模式三（宝妈）+ 模式四（减脂）` 提升品牌势能与复购。
- 成功标准：
  - 周新增成交用户数持续 4 周正增长
  - CAC（获客成本）低于单客毛利的 35%
  - 主力产品 30 日复购率达到 12%+

---

## 1. 约束、假设与决策边界

### 1.1 关键约束
- 团队初期按 3 人最小编制运行（内容/运营/交付）
- 不新增重研发工具，优先模板化和轻交付
- 小红书为主阵地，不做多平台同步扩张

### 1.2 关键假设
- 用户愿意为“可执行系统”付费，而不是仅为“知识信息”付费
- 低价引流款可筛选出高意向用户并有效导向主力款
- 通过周度复盘可持续提升转化效率

### 1.3 失败预警（任一触发即调整）
- 连续 2 周：私信率 < 5%
- 连续 2 周：引流款转化 < 8%
- 连续 2 周：主力款完课率 < 45%

---

## 2. 总体商业系统设计

### 2.1 统一漏斗
1. 触达：诊断型内容吸引精准人群
2. 分流：关键词私信 + 三问问卷
3. 试用：9.9-29.9 引流产品快速成交
4. 放大：99-399 主力产品形成营收主体
5. 升级：499-1999 高客单服务提升利润
6. 留存：复盘社群 + 复购产品包

### 2.2 产品价格带与角色
- 引流款（9.9-29.9）：降低决策门槛，筛选人群
- 主力款（99-399）：标准化交付，规模化营收
- 利润款（499-1999）：个性化解决方案，提高客单价

### 2.3 业务架构图（ASCII）
```text
[内容流量]
    |
    v
[诊断型笔记] --> [评论互动]
    |                 |
    v                 v
[私信关键词] ----> [三问分层]
    |                 |
    v                 v
[引流款成交] ----> [主力款成交] ----> [利润款升级]
    |                                    |
    v                                    v
[打卡/复盘社群] <------------------- [案例回流内容]
```

### 2.4 核心仪表盘（每周必看）
- 流量层：曝光、点击率、收藏率
- 转化层：私信率、引流款转化、主力款转化
- 交付层：完课率、7日留存、30日复购
- 经济层：客单价、毛利率、CAC、LTV

## 3. 模式一：考公/考研冲刺执行系统（现金流引擎A）

### 3.1 用户定义（ICP）
- 人群：备考 2-8 个月、焦虑高、执行不稳
- 痛点：计划常断、复盘缺失、临近考试效率崩盘

### 3.2 产品矩阵
- 引流款：7天冲刺清单（9.9）
- 主力款：21天执行系统（199）
- 利润款：1v1督学（999/21天）

### 3.3 内容引擎
- 诊断类：拖延本质是任务设计错误
- 对比类：同时间不同方法的分数差
- 见证类：7天前后执行数据变化

### 3.4 30天执行
1. 周1：完成 3 套模板与 10 条选题脚本
2. 周2：上线分流帖 + 引流款 + 私信SOP
3. 周3：直播演示“计划到复盘”的完整流程
4. 周4：淘汰后 50% 低效选题，保留高转化池

### 3.5 KPI 与停损
- KPI：私信率 >= 8%，引流款转化 >= 12%，主力款转化 >= 18%
- 停损：主力款转化连续 2 周 < 10%，则重做选题与分流问卷

---

## 4. 模式二：职场副业执行系统（现金流引擎B）

### 4.1 用户定义（ICP）
- 人群：上班族，有副业目标但时间稀缺
- 痛点：方向混乱、执行断续、无阶段结果

### 4.2 产品矩阵
- 引流款：副业时间预算表（9.9）
- 主力款：30天副业推进系统（299）
- 利润款：项目诊断陪跑（1499）

### 4.3 内容引擎
- 框架类：每天 90 分钟推进法
- 纠错类：副业失败的结构性错误
- 案例类：从 0 到首单/首个里程碑

### 4.4 30天执行
1. 周1：完成 3 类行业样板（知识/电商/服务）
2. 周2：连发“真实一周执行记录”
3. 周3：直播拆解真实副业路径
4. 周4：开放限量诊断名额（制造稀缺）

### 4.5 KPI 与停损
- KPI：收藏率 >= 10%，私信到下单 >= 15%，主力款完课 >= 60%
- 停损：完课率连续 2 周 < 45%，则降低任务负荷并重设计交付节奏

---

## 5. 模式三：宝妈家庭节律系统（利润增强引擎）

### 5.1 用户定义（ICP）
- 人群：0-6 岁孩子家庭，家庭协同压力大
- 痛点：早晚高峰混乱、冲突频发、情绪透支

### 5.2 产品矩阵
- 引流款：家庭日程模板（9.9）
- 主力款：21天节律重建包（199）
- 利润款：家庭协同咨询（1299）

### 5.3 内容引擎
- 痛点类：家庭系统失序而非个体懒惰
- 协同类：伴侣分工可执行框架
- 证据类：改造前后日程对比图

### 5.4 30天执行
1. 周1：0-3岁与3-6岁双版本模板
2. 周2：发起 7 天家庭节律挑战
3. 周3：收集UGC并输出案例合辑
4. 周4：咨询SOP标准化并小规模转化

### 5.5 KPI 与停损
- KPI：挑战参与率 >= 20%，UGC提交 >= 15%，咨询转化 >= 5%
- 停损：挑战参与连续 2 周 < 10%，说明话题切口失效，重做问题定义

---

## 6. 模式四：减脂行为坚持系统（复购增强引擎）

### 6.1 用户定义（ICP）
- 人群：久坐职场人，反复减脂失败
- 痛点：难启动、难坚持、缺反馈

### 6.2 产品矩阵
- 引流款：7天启动包（9.9）
- 主力款：30天行为系统（199）
- 利润款：训练+饮食陪跑（999）

### 6.3 内容引擎
- 机制类：失败源于触发器设计错误
- 清单类：20分钟可执行训练流程
- 纠错类：高频无效努力拆解

### 6.4 30天执行
1. 周1：锁定“久坐白领”单一客群
2. 周2：连续发布7天可见变化内容
3. 周3：直播做一次饮食和训练纠偏
4. 周4：上线复购包与陪跑升级包

### 6.5 KPI 与停损
- KPI：7日打卡完成 >= 45%，主力款转化 >= 10%，30日复购 >= 12%
- 停损：7日完成率连续 2 周 < 30%，说明行为设计过重，需降复杂度

---

## 7. 统一运营与交付SOP

### 7.1 周运营节奏
1. 周一：复盘仪表盘 + 选题会 + 资源分配
2. 周二至周六：日更内容 + 私信分流 + AB测试
3. 周日：直播/答疑 + 成交冲刺 + 案例收集

### 7.2 私信标准三问（全模式统一）
1. 你当前最想解决的一个问题是什么？
2. 你每周可投入多少时间？
3. 你希望多久看到第一阶段结果？

### 7.3 交付质量标准
- 每个主力产品必须包含：执行模板、复盘机制、常见失败修复指南
- 每周至少 1 次结构化答疑
- 每期产出可复用案例素材并回流内容端

---

## 8. 财务模型与资源配置（90天）

### 8.1 单客经济（目标值）
- 引流款客单：15 元
- 主力款客单：249 元
- 利润款客单：1199 元
- 综合毛利率目标：>= 65%

### 8.2 结构目标（按成交人数）
- 引流款 : 主力款 : 利润款 = 100 : 25 : 4
- 解释：先放大低价筛选池，再由主力款放大利润款转化

### 8.3 预算原则
- 内容生产投入 <= 当月营收 20%
- 投流预算先小规模测试，ROI 连续两周为正再扩量

---

## 9. 风险、合规与应急预案

### 9.1 合规边界
- 避免医疗诊断与疗效承诺措辞
- 明确“结果与执行程度相关”
- 页面中标注交付边界、服务边界、退款规则

### 9.2 运营风险
- 流量波动：建立备选选题池与二级流量入口（直播/合集）
- 交付过载：触发阈值后限售并延长开班节奏
- 口碑风险：72小时内闭环处理差评并输出修正公告

### 9.3 应急机制
- 任一模式连续 2 周关键指标失真，立即降频并切换到备选模式
- 每月保留 1 次“重构周”，专门修复漏斗短板

---

## 10. 90天路线图（可执行甘特）

### 阶段A（1-30天）：验证期
- 四模式全部跑首轮闭环
- 目标：确认每个模式的最小可行成交路径

### 阶段B（31-60天）：放大期
- 聚焦前两名模式放大内容与转化
- 目标：形成稳定现金流引擎

### 阶段C（61-90天）：固化期
- 复制高转化脚本，建立标准化SOP
- 目标：实现可复制运营，降低个人依赖

---

## 11. 决策建议（最终）
1. 先打透模式一+模式二，优先追求现金流稳定性。
2. 模式三+模式四用于增强复购和高客单，不建议前30天重投入。
3. 每周以“停损阈值”做硬决策，不做情绪化迭代。

2. 保留转化最高的 20% 选题进入复投池。
